# SSH Brute-Force Detection and Automated Response using Wazuh and Shuffle SOAR

## Overview

This project demonstrates an automated Security Operations Center (SOC) workflow for detecting SSH brute-force attacks and performing automated alert enrichment using Wazuh SIEM and Shuffle SOAR.

The solution integrates threat detection, threat intelligence enrichment, and analyst notification to reduce manual effort and improve incident response efficiency.

---

## Project Objectives

* Detect SSH brute-force attacks using Wazuh SIEM
* Forward alerts to Shuffle SOAR via webhook
* Extract attacker IP information
* Enrich alerts using VirusTotal threat intelligence
* Notify SOC analysts of malicious activity
* Demonstrate real-world SIEM and SOAR integration

## Architecture Workflow

1. SSH brute-force attack is simulated from Kali Linux.
2. Wazuh detects multiple failed authentication attempts.
3. Alert is forwarded to Shuffle SOAR using webhook integration.
4. Shuffle extracts the attacker IP address.
5. VirusTotal performs IP reputation analysis.
6. Decision logic determines whether the IP is malicious.
7. Email notification is sent to the SOC analyst when malicious activity is confirmed.

---

## Detection Flow

### SSH Brute-Force Detection

Wazuh correlation rules monitor authentication logs and generate alerts when repeated failed login attempts occur within a defined time period.

### Alert Forwarding

Detected alerts are automatically forwarded to Shuffle SOAR in JSON format for processing and automation.

---

## SOAR Playbook Workflow

### Stage 1 – Webhook Trigger

Receives alerts generated by Wazuh.

### Stage 2 – Alert Parsing

Extracts attacker IP information from the alert payload.

### Stage 3 – Threat Intelligence Enrichment

Queries VirusTotal API to determine the reputation of the source IP address.

### Stage 4 – Decision Logic

Evaluates VirusTotal results and determines whether the IP should be considered malicious.

### Stage 5 – Analyst Notification

Generates an email notification containing attack details and threat intelligence results.

## Testing and Validation

### Test Scenario 1 – SSH Brute-Force Attack

A brute-force SSH attack was simulated from a Kali Linux system using Hydra.

Result:

* Wazuh successfully detected the attack.
* Alert was forwarded to Shuffle SOAR.
* Threat intelligence enrichment completed successfully.

### Test Scenario 2 – Email Notification Verification

Since the test originated from a personal IP address, VirusTotal did not classify it as malicious.

To verify notification functionality, a simulated malicious result was used.

Result:

* Email notification was successfully generated.
* Alert details were delivered to the SOC analyst mailbox.
